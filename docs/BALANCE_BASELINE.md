# Balance Baseline (M0 Prototype)

> 목적: M0 전투/경제 수치의 초기 기준선을 고정해 구현과 튜닝의 출발점을 만든다.

## 1. 전장/세션 기준

- 보드: 5 x 4 (총 20 슬롯)
- 성문 HP: 20
- 세션 길이 목표: 8~12분
- M0 웨이브 수: 5 (5웨이브 보스)

## 2. 전투 기준 수치

- 피해 계산
  - 물리: `final = raw * 100 / (100 + armor)`
  - 마법: `final = raw * 100 / (100 + resist)`
  - 크리: `raw *= critMultiplier`
- 상태이상(초기)
  - `slow`: 2.0s
  - `stun`: 0.8s
  - `burn`: 3.0s

## 3. 경제 기준 수치

- 소환 비용: 3 Gold
- 리롤 비용: 2 Gold
- 골드 수급
  - 웨이브 시작: +3
  - 웨이브 클리어: +5
  - 처치당: +0.5 (평균)

## 4. 티어 확률 곡선

- W1~5: T1 85%, T2 15%, T3 0%
- W6~12: T1 70%, T2 25%, T3 5%
- W13+: T1 55%, T2 35%, T3 10%

## 5. 웨이브 스케일 공식 (초안)

- HP 배율: `hpMul = 1 + 0.12 * (wave - 1)`
- 수량 배율: `countMul = 1 + 0.05 * (wave - 1)`
- M0은 샘플 웨이브(`docs/examples/waves.sample.json`)를 우선 사용하고, 공식은 M1부터 적용

## 6. M0 목표 지표

- W1 클리어율: 95%+
- W5(보스) 도달율: 40~60%
- 평균 런 시간: 6~10분 (프로토타입)
- 치명적 좌절 방지: W1~3에서 무합성 확률 20% 이하

## 7. 측정 로깅 최소 항목

- 웨이브별
  - 시작 골드 / 종료 골드
  - 총 처치 수
  - 누락 적 수(성문 타격)
  - 최고 DPS 유닛
- 런 단위
  - 도달 웨이브
  - 클리어 여부
  - 선택 유물 목록
  - `runSeed`

## 8. 튜닝 우선순위

1. 성문 HP와 웨이브 적 HP 곡선
2. 소환/리롤 비용
3. 티어 확률 곡선
4. 유물 파워(효과값)
