# 06) Rewards & Meta Progression

> 목표: “런 보상 + 계정 성장”을 설계하여, 반복 플레이 동기를 만든다.

## 1. 런(한 판) 안에서의 보상
### 1.1 즉시 보상(전투 중/웨이브 종료)
- 골드: 소환/리롤/강화에 사용(런 종료 시 소멸)
- 회복: 성문 HP 회복(옵션)
- 임시 버프: 다음 웨이브 1회용(예: 공격속도 +15% 1웨이브)

### 1.2 선택형 보상(유물/패시브) — 핵심
- 웨이브 3/6/9/… 종료 시 “3개 중 1개 선택”
- 유물은 “빌드 방향을 바꾸는 힘”이 있어야 한다.
  - 예: “화염 유닛 등장 확률 +20%”
  - 예: “합성 시 10% 확률로 1개 덜 필요(=2개로 합성)”
  - 예: “리롤 비용 고정 1” (경제형)

### 1.3 보스 보상
- 보스 처치 시 “보스 상자” 1개
  - 희귀 유물, 메타 재화(아래), 스킨 조각(추후)

## 2. 런 결과 보상(메타 재화)
### 2.1 메타 재화 제안(가칭)
- `훈장`(Medal): 영구 업그레이드 트리에 사용
- `보급품`(Supply): 도감 해금/유닛 스킨/편의에 사용
- `봉식 EXP`: 계정 레벨(해금/특전)

> MVP: 메타 재화는 1~2개로 단순화 권장(너무 많으면 UX 복잡).

### 2.2 보상 산정(예시)
- 기본: 웨이브 도달 수에 비례
- 추가: 챕터 난이도, 보스 처치, 도전 과제(노-부활, 특정 시너지 클리어 등)

## 3. 영구 업그레이드(메타 성장)
### 3.1 업그레이드 카테고리(예시)
- **훈련**: 시작 공격력/체력 보너스(작게)
- **병참**: 시작 골드 +X, 상점 리롤 비용 -Y
- **성벽**: 성문 HP +X
- **전술**: 시작 유물 1개 선택(고급)
- **행운(주의)**: 희귀 티어 등장 확률 +a%
  - 너무 세면 “과금/성장 없으면 불리” 느낌이 나므로, 상한을 낮게(예: +3~5% 내)

### 3.2 성장의 원칙
- “매우 느린 영구 파워 상승”보다 “선택지(유닛/유물) 확장” 위주가 건강함
- 플레이어가 “내가 똑똑해졌다” 느낌이 들게

## 4. 일일/주간 과제(운영형, MVP 옵션)
- 일일: “3판 플레이”, “시너지 2종 발동”, “보스 1회 처치”
- 주간: “챕터2 클리어”, “특정 빌드로 5회 승리”
- 보상: 보급품/훈장

## 5. 부활/세이브(선택)
- 캐주얼 루트: 1회 “부활” 제공(광고/재화)
- 하드코어 루트: 부활 없음(대신 보상 증가)
- MVP에서는 “부활 없음 + 중단 후 재개(자동 저장)”만으로도 충분

## 6. Codex 산출물
- 보상 테이블(데이터) + UI 플로우
- 업그레이드 트리 데이터 구조 + 적용 시점(런 시작/전투 중)
- 런 결과 정산 로직(메타 재화 계산)
