# 12) Production Plan & Backlog (MVP → v1.0)

> 목표: “어떤 순서로 만들어야 가장 빨리 플레이 가능한가?”를 정한다.

## 1. 마일스톤 정의
### M0 — 프로토타입(플레이 루프 검증)
- 최소 보드/적 스폰/유닛 3종
- 소환/배치/자동 공격
- 웨이브 5개 + 결과 화면

### M1 — MVP(출시 전 최소 기준)
- 유닛 12종 + 시너지 6종 + 유물 20개
- 웨이브 20 + 보스 1
- 로비/도감(간단)/업그레이드(간단)
- 저장/불러오기(중단 후 재개)
- 시드/이벤트 로그 기록

### M2 — 확장(컨텐츠/밸런스)
- 챕터 2 추가
- 적/보스/유물 추가
- 밸런스 시뮬레이터로 자동 튜닝

### M3 — 운영(선택)
- 일일/주간 임무
- 시즌/이벤트
- 스킨/패스

## 2. MVP 백로그(권장 순서)
### 2.1 기반(Foundation)
1) 프로젝트 구조/빌드(웹)
2) 컨텐츠 로더 + 스키마 검증
3) Seeded RNG + runSeed 생성/저장
4) GameCore 전투 루프 최소 구현

### 2.2 전투/런
5) 웨이브 스폰 스케줄러
6) 유닛 배치/합성 로직
7) 시너지 적용
8) 유물 선택/적용
9) 결과 정산 + 런 히스토리 저장

### 2.3 UX
10) 로비/챕터 선택
11) 런 HUD(골드/웨이브/HP/버튼)
12) 툴팁/도감 최소 버전
13) 튜토리얼 1분 버전

### 2.4 품질
14) 단위 테스트(핵심 로직)
15) 로그/디버그 오버레이(개발용)
16) 성능(모바일 30fps 목표)

## 3. 완료 기준(Definition of Done)
- 빌드/실행 가능(로컬)
- 첫 화면 → 플레이 → 결과까지 1판 완주 가능
- 컨텐츠 파일만 수정해도(코드 변경 없이) 유닛/웨이브 일부 수정 가능
- 시드 고정으로 같은 결과 재현 가능

## 4. 리스크 & 대응
- 리스크: “AI 에셋 퀄리티/일관성”
  - 대응: 마스터 캐릭터 고정 + 스프라이트 시트 표준화 + 프레임 흔들림 검수 루틴
- 리스크: “밸런스 폭주”
  - 대응: 시뮬레이터/로그 기반 튜닝(문서 07)

## 5. Codex에게 주는 작업 방식
- 기능 단위 PR로 쪼개기
- 리스크 높은 작업(대량 삭제/DB 변경)은 사전 허가
- 워크스페이스 외 경로 작업은 금지(00 문서)
