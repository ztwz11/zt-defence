# 09) UI / UX Flows

> 목표: “지금 뭐 해야 하지?”가 1초 내로 이해되도록 한다.

## 1. 핵심 UX 원칙
- **정보는 3개만 크게**: (1) 남은 성문 HP (2) 현재 웨이브 (3) 골드
- 나머지는 탭/툴팁으로 숨김
- 모바일 기준: 한 손 조작 가능(버튼 간격)

## 2. 화면 목록(Screens)
### 2.1 로비
- 시작(플레이)
- 업그레이드(메타)
- 도감(유닛/유물)
- 임무(일일/주간)
- 설정(사운드/진동/그래픽)

### 2.2 챕터 선택
- 챕터 카드(난이도/보상/최고 기록)
- 시작 보너스 선택(선택형 1개)

### 2.3 런 HUD(전투 화면)
- 상단: 웨이브 번호 / 타이머(옵션) / 성문 HP
- 하단: 골드 / 소환 버튼 / 리롤 버튼 / (옵션) 배속(1x/2x)
- 우측: 시너지 패널(아이콘 + 카운트)
- 좌측: 유물 슬롯(선택한 유물 아이콘)

### 2.4 보상 선택(유물)
- 3장 카드 UI
- 카드: 이름/효과/희귀도/태그
- “상세 보기(길게 누르기)” + “선택”

### 2.5 결과 화면
- 클리어/실패
- 통계: 도달 웨이브, 총 처치, 최고 DPS 유닛, 시드
- 획득: 메타 재화
- 버튼: 다시 하기 / 로비

## 3. 상호작용(Interaction)
- 유닛 드래그로 이동(슬롯 간 교체)
- 합성은 자동(같은 유닛 3개 있으면 버튼 점등) 또는 드래그로 합성(택1)
- 툴팁:
  - 유닛: 스탯/스킬/태그
  - 시너지: 임계값/효과

## 4. 튜토리얼(최소)
- 1) 소환 3번 → 2) 합성 1회 → 3) 시너지 1회 발동 → 끝

## 5. 접근성/품질
- 색약 대비: 시너지는 색 + 아이콘 + 텍스트 병기
- 진동/효과음 On/Off
- 저사양 모드: 이펙트 간소화

## 6. Codex 산출물
- 화면 상태 정의(라우팅/씬 전환)
- HUD 위젯 목록 + 데이터 바인딩
- 툴팁/카드 UI 컴포넌트 설계(재사용)
